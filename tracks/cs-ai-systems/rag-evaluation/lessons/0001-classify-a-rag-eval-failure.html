Lesson 0001

Classify a RAG Eval Failure

Your first win is diagnostic: given one question, one expected answer, retrieved context, and one model answer, decide which part of the RAG loop failed.

Why this matters

The module mission is to build a small, reproducible evaluation harness over the PMP docs. That harness is only useful if each bad answer points to an engineering action. A bad score should become a specific next experiment: fix retrieval, fix generation, repair the dataset, or inspect both.

Primary source: DeepEval's RAG evaluation guide. It separates retriever metrics such as contextual precision, recall, and relevancy from generator metrics such as answer relevancy and faithfulness. For metric vocabulary, keep the glossary open.

The mental model

Search

The retriever must find the right evidence. If the evidence is absent or buried below weaker chunks, the generator starts from a bad context.

Answer

The generator must use the context correctly. If the evidence is present but the answer contradicts it, the failure is generation.

Record

The dataset must define expected facts and sources. If the case is vague, the system may be judged unfairly.

Worked example

  1. Ask whether the expected fact exists in the corpus.
  2. Check whether the retrieved chunks contain that fact.
  3. Check whether the answer stays faithful to those chunks.
  4. Only then assign the failure label.
Question
What is the first 8-12 week milestone for the program?
Expected fact
Produce a portfolio-grade AI backend system with public evidence around architecture, retrieval or memory, agent/tool boundaries, async workflows, evals, observability, deployment, and written tradeoffs.
Retrieved context 
PROGRAM.md: Within the first 8-12 weeks, produce a portfolio-grade AI backend system with public evidence around architecture, retrieval or memory, agent/tool boundaries, async workflows, evals, observability, deployment, and written tradeoffs.
Actual answer
The first milestone is to choose a module and write its module contract.

The evidence is present. The answer ignores it and gives a stale roadmap-level answer. This is a generation failure, not a retrieval failure.

Practice

Classify this case. Pick the failing part first; the feedback will explain the better label.

Question
What are the artifact levels in the Personal Mastery Program?
Expected facts
Evidence, validated artifact, and public proof.
Retrieved context 
ROADMAP.md: Current phase: Phase 0. Goals: establish repo operating rules, define the first outcome, choose the first module sequence.
CONTEXT.md: A Module is a small /teach workspace with one mission, one module contract, source-backed lessons, and evidence.
Actual answer
The artifact levels are evidence, validated artifact, and public proof.
Choose one label to get feedback.
